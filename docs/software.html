Software

Overview

The Desk Buddy software controls how the robot moves, reacts to sensor input, displays emotions, and responds to user interaction.

The software runs on an STM32F411 microcontroller and is written in C using STM32CubeIDE. The robot uses VL53L0X Time of Flight sensors for edge detection, PWM controlled DC motors for movement, servo motors for ear animation, and an SSD1306 OLED display for facial expressions.

Main Loop

The robot operates using a continuously repeating main loop.

Each iteration of the loop checks the user button, reads sensor data, updates the robot's behavior, controls the motors, moves the ears, and updates the display.

Basic loop:

while robot is running:
check button
read ToF sensors
determine robot mode
control motors
move ears
update display

The robot checks edge detection before normal driving behavior to make sure that it does not fall off.

Software Structure

The software is organized into separate modules to improve readability and simplify debugging.

Software Module Purpose
main.c Main robot behavior and decision making
task_distance.c VL53L0X sensor initialization and distance measurements
motor.c DC motor control using PWM
servo.c Llama ear movement control
screen.c OLED graphics and facial animations

Robot Modes

The robot operates using several behavior modes.

Mode Description
Sleep Stops the motors, lowers the ears, and displays sleeping eyes
Pet Displays happy eyes and wiggles the ears when an object is detected nearby
Wonder Normal operating mode with animated eyes and forward movement
Falling Displays a scared face and turns away from the desk edge

Behavior Logic

The robot uses simple condition based decision making.

First, the user button is checked. If the button is pressed, the robot enters Sleep mode.

The robot then reads the VL53L0X sensors.

If an object is detected very close to the robot, it enters Pet mode. If either sensor detects a large distance which is the edge of a desk, the robot enters Falling mode and turns away from the edge.

Otherwise, the robot enters Wonder mode and continues moving forward.

Motor Control

The robot uses two DC gear motors driven through DRV8833 motor drivers.

Motor speed and direction are controlled using PWM signals generated by the STM32 timers.

Each DC gear motor includes an encoder that measures wheel rotation. The encoder signals are connected to external interrupt pins on the STM32. Each time an encoder pulse is detected, the software updates the left or right encoder count.

The robot uses these encoder counts to estimate wheel speed. A simple closed-loop controller compares the measured wheel speed to a target speed and adjusts the PWM duty cycle sent to each motor. This helps the robot drive straighter because the software can compensate if one wheel is moving faster than the other.

In normal driving mode, the robot continuously updates the motor PWM values using encoder feedback. During edge avoidance, the robot uses a fixed turning command so it can quickly move away from the desk edge.

Sensor Logic

The robot uses VL53L0X Time of Flight sensors to detect nearby objects and desk edges.

The sensors continuously measure distance and provide feedback to the main control loop.

This allows the robot to react to nearby objects while avoiding falls.

OLED Display

An SSD1306 OLED display provides animated facial expressions that help give the robot personality.

The display supports several animations:

  • Normal blinking eyes
  • Sleeping eyes with animated "Z" symbols
  • Happy eyes with floating hearts
  • Scared face for edge detection

The display is updated through the I2C interface.

Ear Servo Control

The Desk Buddy uses two servo motors to move its llama ears.

The ears are used to communicate the robot's emotional state and make interactions feel more natural.

Robot Mode Ear Behavior
Sleep Ears lowered
Pet Ears wiggle continuously
Wonder Ears raised
Falling Ears raised in alert position

The servos are controlled using PWM signals generated by the STM32 timers.
